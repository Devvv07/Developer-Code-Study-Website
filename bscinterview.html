BSc Web Development Interview Questions

Basic Interview Questions

Q1. What is the difference between let, const, and var? Easy

Answer: var is function-scoped, can be redeclared. let is block-scoped, can be reassigned but not redeclared. const is block-scoped, cannot be reassigned or redeclared. Use const by default, let when reassignment needed.

Q2. Explain the difference between == and === in JavaScript. Easy

Answer: == compares values after type coercion (converts types). === compares both value and type without coercion (strict equality). Example: 5 == "5" is true, but 5 === "5" is false.

Q3. What is the difference between localStorage and sessionStorage? Easy

Answer: localStorage persists until explicitly cleared, works across tabs/windows. sessionStorage expires when tab/window closes, isolated to single tab. Both store data as strings, 5-10MB limit.

Q4. What are semantic HTML elements? Easy

Answer: Semantic elements describe meaning to browser and developer (<header>, <nav>, <main>, <article>, <section>, <aside>, <footer>). Improve accessibility, SEO, and code readability compared to generic <div>.

Q5. What is the box model in CSS? Easy

Answer: Every element is a box with content, padding, border, margin. Total width = content + padding + border + margin. Use box-sizing: border-box to include padding/border in width calculation.

Intermediate Questions

Q6. What is CSS specificity and how is it calculated? Medium

Answer: Specificity determines which CSS rule applies when multiple rules target same element. Calculated as: Inline styles (1000) > IDs (100) > Classes/attributes (10) > Elements (1). !important overrides all.

Q7. Explain event bubbling and event capturing. Medium

Answer: Event bubbling: event propagates from target element up to root. Event capturing: event propagates from root down to target. Use addEventListener's third parameter to control: true for capture, false for bubble (default).

Q8. Explain CSS Flexbox vs Grid. When to use each? Medium

Answer: Flexbox is one-dimensional (row OR column), best for components, navigation, aligning items. Grid is two-dimensional (rows AND columns), best for page layouts, complex designs. Can be combined for powerful layouts.

Q9. What are HTTP methods and their uses? Medium

Answer:

  • GET: Retrieve data (idempotent)
  • POST: Create new resource
  • PUT: Update entire resource
  • PATCH: Partial update
  • DELETE: Remove resource

Q10. Explain hoisting in JavaScript. Medium

Answer: Hoisting moves declarations to top of scope during compilation. var declarations hoisted with undefined value. Function declarations fully hoisted. let/const hoisted but in temporal dead zone until initialization. Always declare at top to avoid confusion.

Advanced Questions

Q11. What is closure in JavaScript? Hard

Answer: Closure is when inner function has access to outer function's variables even after outer function returns. Creates private variables, used in callbacks, event handlers, module pattern. Example: function counter with private count variable.

Q12. How does the 'this' keyword work in JavaScript? Hard

Answer: 'this' refers to execution context. In methods: object that owns method. In functions: global object (window). In arrow functions: lexical this (inherits from parent). Can be explicitly set with call(), apply(), bind().

Q13. Explain critical rendering path and optimization. Hard

Answer: Steps: DOM construction ‚Üí CSSOM construction ‚Üí Render tree ‚Üí Layout ‚Üí Paint. Optimize by: minifying CSS/JS, deferring non-critical CSS, async/defer scripts, critical CSS inline, reducing DOM depth, lazy loading images.

Q14. What is CORS and how to handle it? Medium

Answer: CORS (Cross-Origin Resource Sharing) is security mechanism preventing requests from different origins. Handle with: server-side headers (Access-Control-Allow-Origin), JSONP (legacy), proxy server, or using credentials properly.

Q15. Explain JavaScript promises and async/await. Medium

Answer: Promises represent future value, handle async operations. States: pending, fulfilled, rejected. async/await is syntactic sugar for promises, makes async code look synchronous. Use try/catch for error handling.

Coding Interview Questions

Q16. Write a function to reverse a string in JavaScript. Easy

Answer:

function reverseString(str) {
    return str.split('').reverse().join('');
}

console.log(reverseString("hello")); // "olleh"

Q17. Implement a debounce function. Medium

Answer:

function debounce(func, delay) {
    let timeoutId;
    return function(...args) {
        clearTimeout(timeoutId);
        timeoutId = setTimeout(() => func.apply(this, args), delay);
    };
}

// Usage: const debouncedSearch = debounce(searchFunction, 300);

Q18. Write CSS for a responsive navigation bar. Medium

Answer:

/* Basic styles */
nav {
    background: #333;
    color: white;
    padding: 10px 20px;
}

nav a {
    color: white;
    text-decoration: none;
    padding: 10px 15px;
    display: inline-block;
}

/* Responsive styles */
@media (max-width: 600px) {
    nav {
        text-align: center;
    }
    nav a {
        display: block;
        padding: 12px 0;
    }
}

Q19. Create a simple REST API using Node.js and Express. Hard

Answer:

const express = require('express');
const app = express();
app.use(express.json());
app.get('/api/data', (req, res) => {
    res.json({ message: 'Hello World' });
});
app.listen(3000, () => {
    console.log('Server is running on http://localhost:3000');
});
